# 🌌 Vyom — A Story (v0.1 → v0.8)

This is not documentation.  
This is not a changelog.

This is a **story**.

Vyom didn’t start as a programming language.  
It started as a feeling.

> “Easy likhne me ho,  
> par andar se real ho.”

No buzzwords.  
No trends.  
No race.

Bas ek file.  
Ek editor.  
Aur bahut saari galtiyaan.

---

## 🌱 v0.1 — Jab Sirf Ek Line Thi

Sab kuch shuru hua ek line se:

```vy
a = 10
```

No types.  
No scope.  
No rules.

Bas ek sawaal:

**“Kya main kuch yaad rakh sakta hoon?”**

Aur jab us value ko print kiya —  
screen pe number aaya.

Wo moment fake nahi tha.  
Wo moment *real* tha.

Vyom language nahi tha.  
Vyom proof tha —  
ki haan, kuch ban sakta hai.

---

## 🧱 v0.2 — Jab Cheezein Tootni Shuru Hui

Indentation matter karne lagi.  
Whitespace matter karne laga.  
Errors dikhne lage.

Kabhi code chala.  
Kabhi crash hua.

Aur ye achha tha.

Kyuki ab Vyom jhoot nahi bol raha tha.  
Agar kuch galat hai —  
to seedha bol raha tha.

---

## 🧩 v0.3 — Structure Aaya, Bharosa Nahi

Types aaye.  
Rules aaye.

Par execution abhi immature tha.

Yahin samajh aaya:

> Structure banana easy hai.  
> Correct behavior banana hard.

---

## 🔁 v0.4 — Jab Functions Zinda Hue

Functions sirf syntax nahi rahe.  
Wo chale.  
Wo return hue.  
Unka scope bana.

Shadowing ko jaan-bujh kar ban kiya gaya.

Vyom ne pehli baar discipline seekha.

---

## ⚖️ v0.5 — Sach, Jhoot Aur Faisle

Truth define hua:

```
0        → false  
non-zero → true
```

No magic.  
No assumptions.

Vyom ko **guess karna allowed nahi tha**.

---

## 🔄 v0.6 — Jab Repeat Karna Aaya

Loops aaye.  
Testing aayi.  
Aur bahut kuch toota.

Par har bug ke baad Vyom aur strict hota gaya.

---

## 🧠 v0.7 — Discipline Over Features

Ye release flashy nahi tha.

Par important tha.

- `const`
- strict typing
- loop-local scope
- predictable print
- introspection

Vyom ne yahin decide kiya:

> “Main fast nahi banunga.  
> Main correct banunga.”

---

## 🧱 v0.8 — Jab Containers Aaye (Par Control Ke Saath)

Sabse dangerous phase.

Arrays aaye.

Par lists nahi.  
Dynamic resize nahi.  
No freedom without responsibility.

```vy
int a[3] = [1,2,3]
```

- Fixed size  
- Bounds checked  
- Type locked  
- Const respected  

Strings indexable bane —  
par immutable rahe.

```vy
s[0]
```

`len()` aaya.  
`exit()` aaya —  
par memory cleanup ke saath.

Yahin Vyom ne apni philosophy **prove** ki:

> Power sirf tab allowed hai  
> jab control guaranteed ho.

---

## 🌟 Aakhri Baat

Vyom complete nahi hai.  
Par Vyom honest hai.

Aur shayad  
yehi kaafi hai.

—

Vyom ek language nahi.  
Vyom ek process hai.

— Sanket
