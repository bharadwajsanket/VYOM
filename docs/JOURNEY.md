# 🌌 Vyom — The Journey (0.1 → 0.5)

This is not documentation.  
This is memory.

Vyom did not start as a language.  
It started as a thought:

> “Python jaisa easy ho,  
> par andar se C jaisa real ho.”

No buzzwords. No hype.  
Bas ek file, ek idea, aur bahut saare mistakes.

---

## 🌱 v0.1 — The First Line

It began with one line:

```
a = 10
```

No types.  
No rules.  
No future planning.

Sirf ye dekhna tha:
**“Can I store something and print it?”**

And when it printed…  
that feeling was real.

---

## 🧱 v0.2 — Things Started Breaking

Then came:
```
print a
```

Indentation mattered.  
Errors started appearing.

Sometimes code ran.  
Sometimes it crashed.

And that was good.

Because now Vyom was **honest**.  
If something was wrong, it screamed.

---

## 🧩 v0.3 — Structure Without Confidence

This version tried to be mature.

```
int a = 10
a = 1
```

Types were *optional*.  
Locking existed, but execution didn’t trust itself yet.

Functions were parsed, not run.  
Like promises not yet fulfilled.

This phase taught one thing clearly:

> Structure is easy.  
> Correct behavior is hard.

---

## 🔁 v0.4 — Functions Finally Breathed

This was a turning point.

Functions were no longer fake.  
They ran.  
They returned.  
They had scope.

```
def add(a, b):
    return a + b
```

Local vs global mattered now.  
Shadowing was **banned** — on purpose.

Many bugs came here.  
Stack bugs.  
Return bugs.  
Scope bugs.

Each fix made Vyom more real.

---

## ⚖️ v0.5 — Logic, Truth, and Control

This version asked a dangerous question:

> “What is true?”

Answer:
```
0 = false
non-zero = true
```

No `true` / `false` keywords.  
No magic.

Then came comparisons:
```
a < b
a == b
```

Then logic:
```
and
or
not
```

And finally… control flow:

```
if
elif
else
```

This part broke **everything** at least once.

- elif without if  
- empty expressions  
- blocks executing twice  
- returns inside conditionals  

But one by one, each issue was faced.  
Not ignored.  
Not patched blindly.

**Understood. Then fixed.**

---

## 🛠️ The Installer Phase (Painful but Necessary)

Windows taught a brutal lesson:

> “Your program is useless if people can’t run it.”

PATH issues.  
Admin permissions.  
Terminal restarts.

So:
- install.bat was born  
- then fixed  
- then fixed again  
- then made safer  

Same with uninstall.bat.

Not fun.  
But real.

---

## 🧠 What Vyom Became

By v0.5, Vyom learned to:

- execute line by line  
- manage its own call stack  
- respect scope  
- evaluate expressions correctly  
- say **NO** clearly when code is wrong  

It still doesn’t do much.

And that’s the point.

---

## 🧭 What This Journey Taught

- Speed is not syntax  
- Simplicity is not weakness  
- Errors are teachers  
- Features added too early rot the design  

Most importantly:

> A language is not built by adding features  
> It is built by **saying no** to many of them.

---

## ✨ Final Note

Vyom is not finished.  
It is **becoming**.

And this file exists so one day,  
when the code looks clean and confident…

You remember:

It was once messy.  
And that was necessary.

— Sanket
